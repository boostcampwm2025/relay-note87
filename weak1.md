## ✅ 선택한 퀘스트: 2️⃣ AI에게 받은 답변을 AI를 통해 크로스 체킹 해보기

- **Who**: 윤장호, 박혜린, 이석우, 이세준
- **사용 AI**: 퍼플렉시티, 클로드
- **선택 배경/이유**: 다양한 AI 사용 경험을 늘리고자 함
- **달성 기준**: 개인적으로 느낀 차이점 1회 기록하여 정리하기

### 💬 세준의 느낀 점

- **퍼플렉시티**는 검색 기반 응답이라 **정보 탐색에 강점**이 있었다. 실제 논문이나 웹 출처를 제시하며 객관적인 사실을 빠르게 찾을 수 있었다.
- **클로드**는 대화체가 부드럽고 **설명과 정리에 강점**이 있었다. 맥락을 이해하고 친절하게 말해줘서 학습에 도움이 되는 느낌을 받았다.
- 예시 질문에 대해 퍼플렉시티는 "출처 A에 따르면…" 식으로 설명했지만, 클로드는 "이렇게 생각해볼 수 있어요"처럼 접근했다.
- **요약**:

  - 퍼플렉시티 → 빠른 정보 확인에 유용
  - 클로드 → 깊이 있는 이해와 해석에 유용

### 💬 장호의 느낀 점

- 평소의 써보지 않았던 퍼플렉시티를 사용해보았습니다!
- SKT의 에이닷과의 프로모션, 학생 인증 등으로 Pro를 사용해볼 수 있었습니다.
- 장점
  - Sonar, GPT, Claude, Gemini 등의 모델 중에서 입력 받은 쿼리에 대해 최선의 모델을 선택 후 결과를 도출해주기 때문에, 항상 최선의 결과를 받아볼 수 있었습니다.
  - 답변에 대한 출처가 함께 제공되기 때문에 신뢰도가 높다고 느꼈고, 추가적인 학습에 사용할 수도 있었습니다.
- 단점
  - UI가 조금 딱딱한 느낌이 들었습니다.
  - 여러 출처에서 정보를 가져오다보니 다른 생성형 AI에 비해 프롬프트가 출력되는 속도가 느리다고 느꼈던 것 같습니다.
  - 쿼리마다 최선의 모델을 선택해서 그런건지 바로 이전의 내용을 잘 기억하지 못 하고 문맥을 파악하지 못 하는 모습을 보였던 적이 있었습니다.

### 💬 혜린의 느낀 점
- 학습 중에 같은 내용에 대해서 퍼플렉시티와 클로드의 답변이 어떻게 다를지 궁금했다.  
- 프롬프트  
```
이벤트 시스템 설계 원리에 대해서 공부하고 싶어, Publisher-Subscriber 패턴을 언제 사용하는지, 어떤 구현 방식이 있는지를 중심으로 설명해주고 EventManager에 대해서 설명해줘
```
- **퍼플렉시티**  
    - 참고한 웹 페이지들의 출처가 20개 가량 함께 제시되어서 정보에 신뢰가 갔음  
- **클로드**  
    - 코드를 구현해달라는 의도는 아니였는데 Publisher-Subscriber 패턴의 주요 구현 방식에 대해 설명해달라고 하니까 직접 코드를 구현해서 설명  
    - 퍼플렉시티보다 코드 제외하고도 설명 내용이 좀 더 길고 자세히 설명되어 있었음    

- **느낀점**  
    - 퍼플렉시티는 출처가 제공되어서 신뢰도가 있다고 느껴졌다.  출처를 직접 확인해보면서 퍼플렉시티가 설명해준 내용만 받아들이지 않고 좀 더 적극적으로 공부를 할 수 있게 한다는 점에서 좋았다.    
    - 클로드는 코드를 통해서 자세히 설명하려는 느낌이 있었고 퍼플렉시티가 설명하지 않는 부분들에 대해서도 설명해주었다. 그러나 주요 구현 방식으로 4가지를 설명했는데 그 중에서 코드로 직접 구현해준 것은 1번 뿐이라서 설명에 일관성이 좀 부족하다는 느낌을 받았다.

## ✅ 선택한 퀘스트: 4️⃣ 오늘 하루 고생한 나에게 AI를 통해 위로 받아보기

### 💬 세준의 느낀 점

- **라일리 하루**는 단순 분석을 넘어서 **정서적 지지를 주는 대화형 AI**로, 감정을 공감해주는 방식이 인상 깊었다.
- 예를 들어, "오늘 너무 지치고 뭘 해도 안 풀리는 기분"이라고 적었을 때,  
  하루는 "그럴 수 있어요. 세준님이 너무 열심히 해와서 그런 걸 수도 있어요"라고 따뜻하게 받아주었고,  
  **산책이나 짧은 글쓰기**를 제안했다.
- 실제로 산책을 해보니 기분이 환기되고, 감정 일기 작성이 스트레스 해소에 실질적인 도움이 되었다.
- **요약**:
  - 감정 정리에 매우 유용했고 자기 자신을 이해하고 돌보는 습관 형성에 기여했다.

### 💬 장호의 느낀 점

- 장점
  - 특정 이벤트마다 제 감정을 기록하는 과정에서, 기분이 좀 정돈되는 기분을 받았던 것 같습니다. 적은 내용을 바탕으로 AI가 제가 느끼는 감정을 글로 정리해주는 것도 좋았던 것 같습니다.
  - 체크인을 까먹고 못 했을 때 ‘매우 슬퍼요’ 감정을 기록했을 때 알림으로 ‘“장호”님의 감정상태가 좋지 않습니다.’를 보내주었습니다. 저도 알고 있는데 뭔가 비상인 것처럼 저 대신 오버 리액션을 해주니 기분이 좀 풀렸던 것 같습니다.
- 단점
  - 하루에 1~2개씩 감정적인 변화가 생길 때마다 적었는데, 너무 적어서 그런지 아직 제 감정 상태에 대한 리포트를 받아보지 못 했습니다.
  - AI로 감정 판단을 받은 뒤, 내용을 수정하고 싶어서 뒤로 갔다가 다시 진행하면 똑같은 내용으로 중복된 감정 상태가 기록되는 버그가 있었습니다.
